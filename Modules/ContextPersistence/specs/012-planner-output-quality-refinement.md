# Execution Spec: 012-planner-output-quality-refinement

## Feature
- context-persistence

## Purpose
- Ensure execution specs generated by the planner are clear, actionable, and non-tautological.
- Replace vague or self-referential planning output with deterministic, meaningful implementation steps derived from canonical context.

## Scope
Refine the behavior of the execution spec planner so that it produces:
- High-quality execution spec metadata:
  - Purpose
  - Scope
  - Requested Changes
  - Slug (spec id suffix)
- Deterministic, bounded planning output based on:
  - canonical feature spec
  - feature state
  - decision ledger

## Constraints
- Canonical feature context remains the only source of truth.
- Planning must remain deterministic, reproducible, and bounded to a single step.
- No LLM usage.
- No mutation of canonical feature documents.
- No expansion into multi-step planning.
- Must reuse existing parsing / inspection / gap-detection mechanisms where possible.

## Requested Changes

### 1. Gap Detection Must Be Concrete
Planner must derive differences between:
- Expected behavior (feature spec)
- Current implementation state (feature state doc)

### 2. Requested Changes Must Be Actionable
Must describe concrete implementation steps.
Avoid vague phrases like:
- "Improve planning quality"
- "Ensure better alignment"

### 3. Purpose Must Explain Timing
Explain why this step exists now.

### 4. Scope Must Be System-Oriented
Define what part of the system is being advanced.

### 5. Slug Generation Must Be Meaningful
Avoid vague verbs like:
- keep, support, ensure, improve

### 6. Tautology Detection
Reject outputs where sections are redundant.

### 7. Block When No Meaningful Step Exists
Return status=blocked with required_actions.

## Non-Goals
- No LLM-based planning
- No multi-step planning
- No modification of canonical docs

- Non-tautological output
- Actionable specs
- Deterministic output
- All tests pass
