# Execution Spec: 007.002-agent-facing-doc-path-contracts

## Feature

- execution-spec-system

## Purpose

Align all agent-facing Foundry instruction surfaces with the canonical feature documentation layout.

This spec repairs stale guidance created during earlier modular-docs work, including any incorrect references to feature context files named as `<id>-<slug>.*.md`, any direct `docs/<feature>/` layout references, and any remaining `docs/specs/` references except historical migration notes.

## Background

`007-modular-docs-feature-layout` introduced modular feature documentation. `007.001-correct-feature-docs-layout` may already have been implemented early in this repository, moving the final canonical layout under `docs/features/<feature>/`.

This spec must therefore be safe to run after `009`. It must not try to move feature directories again. Its job is to make agent-facing instructions match the final layout now present in the repository.

## Depends On

- `007-modular-docs-feature-layout.md`

## Canonical Path Contract

Feature context files are feature-local and named after the feature:

```text
docs/features/<feature>/<feature>.spec.md
docs/features/<feature>/<feature>.md
docs/features/<feature>/<feature>.decisions.md
```

Feature execution specs are feature-local:

```text
docs/features/<feature>/specs/<id>-<slug>.md
docs/features/<feature>/specs/drafts/<id>-<slug>.md
```

Feature implementation plans are feature-local:

```text
docs/features/<feature>/plans/<id>-<slug>.md
```

Feature support files are:

```text
docs/features/README.md
docs/features/implementation-log.md
```

## Invalid Active Path Patterns

The following patterns must not appear as active instructions:

```text
docs/<feature>/
docs/specs/<feature>/
docs/specs/implementation-log.md
docs/specs/README.md
docs/features/<feature>/<id>-<slug>.spec.md
docs/features/<feature>/<id>-<slug>.md
docs/features/<feature>/<id>-<slug>.decisions.md
docs/features/<id>-<slug>.spec.md
docs/features/<id>-<slug>.md
docs/features/<id>-<slug>.decisions.md
```

They may appear only inside explicitly labeled historical migration notes or tests that intentionally assert invalid-path failures.

## Scope

Update agent-facing instruction surfaces, including at minimum:

- `AGENTS.md`
- `APP-AGENTS.md`
- scaffolded agent instruction stubs
- repository skill files and skill templates, if present
- CLI/help text intended for agents
- documentation sections that instruct agents where feature specs, state files, decision ledgers, execution specs, draft specs, implementation plans, or implementation logs live

## Required Rules

1. Agent-facing instructions must use the canonical path contract exactly.
2. Feature context files must use `<feature>` as their filename stem, not `<id>-<slug>`.
3. Execution specs must remain under `docs/features/<feature>/specs/`.
4. Draft execution specs must remain under `docs/features/<feature>/specs/drafts/`.
5. Implementation plans must live under `docs/features/<feature>/plans/`.
6. The implementation log must be referenced only as `docs/features/implementation-log.md`.
7. No active instruction may describe `docs/specs/` as canonical.
8. No active instruction may describe feature folders directly under `docs/` as canonical.
9. Historical references may remain only when explicitly labeled as historical or migration-only.
10. Do not alter unrelated docs directories such as `docs/templates/`, `docs/versions/`, `docs/whitepapers/`, `docs/policies/`, `docs/philosophy/`, or `docs/architecture/`.

## Implementation Steps

1. Search agent-facing files for stale path guidance:
   - `docs/<feature>/`
   - `docs/specs`
   - `docs/features/<id>-<slug>`
   - `docs/features/<feature>/<id>-<slug>`
   - `docs/features/*`
   - `docs/features/*/specs/*`
   - `implementation-log.md`
   - `plans/`
2. Replace stale active guidance with the canonical path contract.
3. Ensure `AGENTS.md` contains an exact source-of-truth section equivalent to:
   - `docs/features/<feature>/<feature>.spec.md` → authoritative feature intent
   - `docs/features/<feature>/<feature>.md` → current state
   - `docs/features/<feature>/<feature>.decisions.md` → append-only decision history
   - `docs/features/<feature>/specs/*.md` → execution specs
   - `docs/features/<feature>/specs/drafts/*.md` → draft execution specs
   - `docs/features/<feature>/plans/*.md` → implementation plans
   - `docs/features/implementation-log.md` → completed execution-spec ledger
4. Update APP-level instructions separately from framework-level instructions.
5. Update scaffolded agent files so newly initialized apps receive the corrected guidance.
6. Preserve deterministic, frozen-contract, coverage, verification, and implementation-log requirements.
7. Do not move files unless an agent-facing file itself is in a wrong location.
8. Do not create fallback behavior for old paths.

## Acceptance Criteria

- `AGENTS.md` describes only the canonical feature docs layout as active guidance.
- `APP-AGENTS.md` describes only the canonical feature docs layout where framework paths are relevant.
- Scaffolded agent instruction files contain only canonical active path guidance.
- Repository skills and skill templates contain only canonical active path guidance.
- No active instruction says feature context files use `<id>-<slug>` filename stems.
- No active instruction says `docs/specs/` is canonical.
- No active instruction says feature folders live directly under `docs/`.
- References to `docs/features/implementation-log.md` are preserved and correct.
- References to implementation plans use `docs/features/<feature>/plans/<id>-<slug>.md`.
- Any remaining stale path references are clearly historical, migration-only, or invalid-path test fixtures.

## Verification

Run:

```bash
php bin/foundry spec:validate --json
php bin/foundry verify context --json
php bin/foundry verify contracts --json
php bin/foundry verify graph --json
php bin/foundry verify pipeline --json
php vendor/bin/phpunit
php -d xdebug.mode=coverage vendor/bin/phpunit --coverage-text
```

Also run text searches and inspect all remaining matches:

```bash
grep -R "docs/<feature>" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
grep -R "docs/specs" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
grep -R "docs/features/<id>-<slug>" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
grep -R "docs/features/<feature>/<id>-<slug>" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
```

Any remaining matches must be removed, corrected, or explicitly labeled as historical or invalid-path test coverage.

## Implementation Log

Append the completion entry to `docs/features/implementation-log.md` only after all verification commands pass.
