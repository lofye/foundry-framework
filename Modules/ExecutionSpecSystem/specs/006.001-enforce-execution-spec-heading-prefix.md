# Execution Spec: 006.001-enforce-execution-spec-heading-prefix

Problem:
Execution specs are sometimes created with headings like:

`# 001-example`

instead of the required canonical form:

`# Execution Spec: 001-example`

Requirements:
1. Update the draft execution spec stub so spec:new always creates:
`# Execution Spec: <id>-<slug>`

2. Update spec:validate so execution specs fail validation unless their first heading exactly matches:
`# Execution Spec: <filename-without-.md>`

3. Ensure spec:validate reports a deterministic error including:
- file path
- expected heading
- actual heading

4. Update docs/features/README.md to state the required heading format.

5. Update AGENTS.md to explicitly forbid filename-only headings and require:
`# Execution Spec: <id>-<slug>`

6. Add tests proving:
- valid heading passes
- filename-only heading fails
- malformed prefix fails
- hierarchical IDs pass, e.g.
`# Execution Spec: 015.002.001-grandchild`
