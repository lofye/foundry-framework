# Framework Capabilities

Foundry capabilities are organized by architecture area so developers can reason about behavior without internal build-history terminology.

## Original goals

Foundry started with a goal to make framework behavior explicit enough for AI-assisted development without hiding runtime truth from humans.

## Compiler and graph foundations

Foundry establishes semantic compilation, graph inspectability, architecture diagnostics, visualization, and AI prompt-loop foundations.

## Runtime indexing and verification

Foundry strengthens generated runtime indexes, verification workflows, and execution model inspectability for deterministic behavior.

## Application scaffolding capabilities

Foundry includes resource, starter, admin, upload, and listing capabilities through extension packs and explicit feature contracts.

## API and notification capabilities

Foundry includes notification workflows, API/OpenAPI surfaces, docs generation support, and deeper testing workflows.

## Platform services capabilities

Foundry includes billing, workflows, orchestration, search, streams, localization, roles/policies, and inspect UI capabilities through graph-native extension packs.

These capabilities compose through shared compiler, graph, pipeline, and verification infrastructure.
